Geometry Picker

Geometry Picker geometrically generates evenly-spaced particle positions and orientations for subtomogram averaging by sampling a shape fitted to a set of markers (or an arbitrary surface mesh). The result is a TOM/AV3 motive list that is displayed live with Place Object and can be saved as EM (.em) or STOPGAP (.star). This is a ChimeraX port of K. Qu's Pick Particle plugin (Qu K. et al. 2021, Science 373:700–704), with added Filament and Surface modes.

Open the tool from Tools → Volume Data → Geopickr; these controls are on the Geometry Picker tab, alongside the Place Points and Place Object tabs.

Workflow

  1. Place markers with Place Points (or open a .cmm): one marker per sphere center, or an ordered set of markers along a tube/filament axis.
  2. Select the marker set(s), choose an Object style, set the radius and sampling, and click Pick.

Object styles

  • Sphere – each marker is a sphere center; particles are sampled over the sphere surface, oriented along the outward normal.
  • Tube – each marker set defines a tube axis (a natural cubic spline through ≥2 markers); particles are sampled on the tube surface at the given radius, oriented along the surface normal.
  • Filament – particles are placed on the spline centerline with Z along the local tangent; an optional Twist adds a helical in-plane rotation per step.
  • Surface – particles are distributed over any open surface mesh (OBJ/PLY/STL opened in ChimeraX) with approximately even spacing, oriented along the mesh normal.

Sampling

  • Tangential – spacing across the surface (sphere/tube/surface).
  • Axial – spacing along the axis (tube/filament).
  • Twist – helical angle per step (filament).
  • Randomize phi – randomize the in-plane angle (recommended for spheres/surfaces where it is undefined).
  • Tomogram ID – stamped into the motive list for downstream extraction.

Preview / Pick / Save

  • Preview fit draws the fitted sphere/tube as a wireframe so you can match the radius to the feature; Clear preview removes it.
  • Pick generates the particles and shows them immediately in Place Object.
  • Save… writes the motive list as .em or STOPGAP .star.

Utilities

  • Combine – merge several .em motive lists (glob), optionally renumbering particles.
  • Shift – translate particles by a (rotated) offset and/or rescale between pixel sizes.

Command

pickparticle markers [style sphere|tube|filament|surface] [radius r] [tangential t] [axial a] [twist deg] [randomPhi true|false] [tomoId n] [onSurface surface-model]

open spheres.cmm
pickparticle #2 style sphere radius 50 tangential 10

Exporting particles

The Export… button (in the Geometry Picker and Place Object tabs) writes the particle list to several subtomogram-averaging formats:

  • EM motive list (TOM/AV3 .em) and STOPGAP .star.
  • Dynamo .tbl (35-column table), with an optional Dynamo volume list (.vll) to append a > table.tbl reference for geometry-assisted cropping/STA.
  • RELION 5.1 particles star (centered-Ångström coordinates, orientation in rlnTomoSubtomogram*) and RELION 3/4 tomo star (pixel coordinates, rlnAngleRot/Tilt/Psi).

Dynamo and RELION need voxel/Å coordinates, while Geopickr particles are in ChimeraX scene units — choose the source Tomogram (a Volume) in the dialog so voxel size, box dimensions and origin are read from the map and coordinates are converted correctly. Without a Volume, coordinates are assumed to already be voxels.

Note: the Dynamo/RELION conventions are a beta feature — validate against a real import and report issues with an example file.

Command (export)

geopickr export model file path [format em|stopgap|dynamoTbl|relion5|relion3] [onTomogram volume] [tomoId n] [tomoName name] [vll path]

geopickr export #3 file particles.tbl format dynamoTbl onTomogram #1 vll tomograms.vll
